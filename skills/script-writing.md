# 비디오 스크립트 작성 가이드

## 개요

영상 스크립트를 체계적으로 작성하는 규칙과 패턴. 스크립트는 영상 제작의 뼈대이므로
나레이션, 비주얼 힌트, 씬 구분을 명확히 분리하여 작성한다.

---

## 비디오 유형별 구조

### Tutorial (튜토리얼)

시청자가 특정 기술이나 방법을 배우도록 돕는 영상.

- **인트로 (15-30초)**: 문제 제시 + 결과 미리보기 (완성 데모)
- **본문**: 단계별 설명 (Step 1, Step 2, Step 3...)
  - 각 단계는 독립된 씬으로 분리
  - 시청자가 따라 할 수 있도록 속도 조절
  - 실수 포인트나 팁을 씬 내에 포함
- **아웃트로 (15-30초)**: 요약 + CTA (구독, 다음 영상 안내)

**예시 흐름**
```
씬 01: 인트로 - "이 영상을 보면 React Hook을 완전히 이해할 수 있습니다"
씬 02: 사전 준비 - 필요한 도구, 환경 설정
씬 03: Step 1 - useState 기초
씬 04: Step 2 - useEffect 이해
씬 05: Step 3 - 커스텀 Hook 만들기
씬 06: 아웃트로 - 요약 + CTA
```

---

### Explainer (설명 영상)

개념이나 아이디어를 명확하게 설명하는 영상.

- **훅 (10-15초)**: 흥미로운 질문 또는 충격적인 통계
- **배경**: 왜 이 주제가 중요한지 컨텍스트 제공
- **핵심 개념**: 3-5개 포인트 (한 씬에 하나씩)
- **결론 + CTA**: 핵심 정리 + 행동 유도

**예시 흐름**
```
씬 01: 훅 - "왜 대부분의 개발자가 TypeScript를 잘못 사용할까요?"
씬 02: 배경 - TypeScript의 현재 위치와 중요성
씬 03: 포인트 1 - any 타입 남용 문제
씬 04: 포인트 2 - 타입 추론 활용법
씬 05: 포인트 3 - 제네릭의 올바른 사용
씬 06: 결론 + CTA
```

---

### Motion Graphics (모션그래픽)

데이터, 통계, 프로세스를 시각적으로 표현하는 영상.

- **비주얼 중심**: 나레이션은 보조 역할
- **씬당 텍스트 최소화**: 10-15단어 이내
- **데이터 시각화 강조**: 차트, 그래프, 인포그래픽
- **트랜지션 중요**: 씬 간 자연스러운 연결

**예시 흐름**
```
씬 01: 타이틀 카드 + 배경음악
씬 02: 핵심 통계 (큰 숫자 + 맥락)
씬 03: 데이터 차트 (성장 곡선)
씬 04: 비교 인포그래픽
씬 05: 결론 텍스트
```

---

### Review (리뷰)

제품, 서비스, 도구를 분석하고 평가하는 영상.

- **소개 + 첫인상**: 개봉/설치 경험, 외관
- **기능별 분석**: 각 주요 기능을 씬 단위로 분리
- **장단점 비교**: 시각적 비교 테이블 또는 목록
- **종합 평점 + 추천**: 점수 + 어떤 사람에게 추천하는지

**예시 흐름**
```
씬 01: 인트로 - 제품 소개
씬 02: 설치/설정 경험
씬 03: 기능 A 분석
씬 04: 기능 B 분석
씬 05: 장단점 비교
씬 06: 종합 평점 + 추천 대상
```

## 씬 분할 규칙

- **하나의 씬 = 하나의 핵심 메시지** (한 씬에 여러 개념 금지)
- **씬당 권장 길이**: 15-60초
- **최소 씬 길이**: 5초 (너무 짧으면 시청자가 따라가기 어려움)
- **최대 씬 길이**: 90초 (너무 길면 집중력 저하)
- **전체 영상 길이 계산**: 개별 씬 길이 합계 ± 10%
- **각 씬에 [VISUAL] 힌트 필수 포함**

### 씬 번호 규칙

```
씬 01, 씬 02, 씬 03 ... (두 자리 숫자)
파일명: script-01.md, script-02.md
```

## 워드카운트 가이드

나레이션 텍스트 분량으로 영상 길이를 추정할 때 사용한다.

| 영상 길이 | 한국어 글자 수 | 영어 단어 수 |
|-----------|---------------|-------------|
| 30초 | 125-150자 | 65-75 |
| 1분 | 250-300자 | 130-150 |
| 2분 | 500-600자 | 260-300 |
| 5분 | 1,250-1,500자 | 650-750 |
| 10분 | 2,500-3,000자 | 1,300-1,500 |
| 15분 | 3,750-4,500자 | 1,950-2,250 |

**계산 공식**

```
예상 영상 길이(초) = 총 글자 수 ÷ 4.5  (한국어, 250자/분 기준)
예상 영상 길이(초) = 총 단어 수 ÷ 2.3  (영어, 140 words/min 기준)
```

## 톤 가이드

콘텐츠 성격에 따라 나레이션 톤을 선택한다.

### Professional (전문가)
- **문체**: 경어체 (습니다/입니다)
- **특징**: 데이터 중심, 정확한 수치, 학술적 표현
- **예시**: "TypeScript의 타입 시스템은 컴파일 타임에 오류를 감지합니다"

### Casual (친근함)
- **문체**: 해요체 또는 반말
- **특징**: 비유 활용, 일상적 언어, 공감대 형성
- **예시**: "사실 저도 처음에 이 개념이 완전 헷갈렸어요"

### Educational (교육적)
- **문체**: 설명체, 질문-답변 구조
- **특징**: 단계적 설명, 예시 중심, 반복 강조
- **예시**: "먼저 A를 이해하면, 그 다음에 B가 자연스럽게 따라옵니다"

### Energetic (에너지틱)
- **문체**: 짧은 문장, 감탄사, 강조
- **특징**: 빠른 속도감, 강한 훅, 흥분 유발
- **예시**: "믿기지 않죠? 단 10줄로 이게 가능합니다!"

## 스크립트 파일 포맷

### 씬별 파일: `scripts/script-{N}.md`

```markdown
# 씬 {N}: {제목}

## 메타 정보

- 예상 길이: {초}s
- 나레이션 글자 수: {N}자
- 비주얼 유형: [코드데모|애니메이션|텍스트|화면녹화|모션그래픽]
- 씬 유형: [인트로|본문|아웃트로|전환]

## 나레이션

{나레이션 텍스트를 여기에 작성한다.
자연스럽게 읽히도록 구어체로 작성한다.}

## 비주얼 노트

[VISUAL] {화면에 보여줄 것 — 코드, 이미지, 다이어그램 등}
[TRANSITION] {이전 씬에서 이 씬으로의 전환 효과 — fadeIn, slideFromLeft 등}
[TEXT-ON-SCREEN] {화면에 오버레이할 텍스트}
[HIGHLIGHT] {강조할 요소}

## 키 메시지

- {이 씬을 보고 시청자가 얻어가야 할 핵심 1가지}
```

## 나레이션 작성 팁

### 읽기 좋은 문장 작성

- 한 문장은 30자 이내로 짧게 끊는다
- 주어-동사-목적어 순서를 지킨다
- 수동태보다 능동태를 사용한다
- 전문 용어 첫 등장 시 괄호로 영문 병기 (예: 타입 추론(Type Inference))

### TTS 사용 시 주의사항

- 발음하기 어려운 약어는 풀어서 작성 (예: "API" → "에이피아이" 또는 풀어쓰기)
- 숫자는 읽히는 대로 작성 (예: "1000" → "천" 또는 "일천")
- 특수문자 제거 (TTS가 잘못 읽을 수 있음)
- 영어 단어 혼용 시 읽기 방식 확인

### 강조와 일시정지

- 중요한 개념 앞에 "..." 또는 SSML `<break>` 태그로 일시정지
- 핵심 단어는 [HIGHLIGHT] 태그로 비주얼에서 강조 표현

---

## 주의사항

- 나레이션과 비주얼 힌트를 반드시 분리하여 작성한다
- 매 씬마다 [VISUAL] 태그 포함 필수
- 전체 영상 길이 = 개별 씬 길이 합계 ± 10%
- TTS 사용 시 발음하기 어려운 약어는 풀어서 작성한다
- 씬 번호는 두 자리 숫자로 통일한다 (01, 02, ... 09, 10)
- 나레이션만 읽어도 내용이 완전히 전달되어야 한다 (오디오만 들어도 이해 가능)
- 비주얼은 나레이션을 보완하는 역할이지, 대체하는 역할이 아니다
