# 스토리보드 설계 규칙

## 개요

스크립트를 시각적 스토리보드로 변환하는 규칙. 스토리보드는 Remotion 컴포넌트 구현 전
각 씬의 레이아웃, 애니메이션, 에셋을 시각적으로 계획하는 단계다.

---

## 스토리보드 작성 흐름

```
scripts/script-{N}.md       →    storyboard/scene-{N}.md
[나레이션 + VISUAL 힌트]          [레이아웃 + 프레임 계산 + 에셋 목록]
         ↓
storyboard/overview.md
[전체 타임라인 + 일관성 체크]
```

---

## 씬별 레이아웃 테이블 포맷

각 씬의 시각 구성을 시간 흐름에 따라 정리한다.

| 시간(s) | 프레임 | 나레이션 | 화면 구성 | 애니메이션 | 에셋 |
|---------|--------|---------|----------|-----------|------|
| 0-2s | 0-60 | (없음) | 배경 + 로고 | fadeIn | logo.svg |
| 2-5s | 60-150 | "안녕하세요..." | 텍스트 타이틀 | slideFromBottom | - |
| 5-15s | 150-450 | "오늘은..." | 코드 블록 | typewriter | code-01.ts |

## 프레임 계산

모든 시간 단위는 프레임(frame)으로 관리한다. 초(second)로 혼용하지 않는다.

```
FPS: 30 (기본값, constants.ts에서 관리)

durationInFrames = 초(s) × FPS
예: 5초 씬 → durationInFrames = 5 × 30 = 150

씬 시작 프레임 = 이전 모든 씬의 durationInFrames 합계
예:
  씬 01: from=0,   durationInFrames=150  (0s ~ 5s)
  씬 02: from=150, durationInFrames=300  (5s ~ 15s)
  씬 03: from=450, durationInFrames=200  (15s ~ 21.7s)

전체 영상 프레임 = 모든 씬 durationInFrames 합계
```

### 프레임 계산 테이블 예시

| 씬 | 길이(s) | durationInFrames | from 프레임 | to 프레임 |
|----|---------|-----------------|------------|---------|
| 01 | 5s | 150 | 0 | 150 |
| 02 | 10s | 300 | 150 | 450 |
| 03 | 8s | 240 | 450 | 690 |
| **합계** | **23s** | **690** | - | - |

## 트랜지션 패턴 카탈로그

씬 간 전환에 사용하는 표준 트랜지션 목록.

| 이름 | 설명 | 권장 상황 | 프레임 수 |
|------|------|----------|---------|
| `fadeIn` | opacity 0 → 1 | 씬 시작, 텍스트 등장 | 15-30f |
| `fadeOut` | opacity 1 → 0 | 씬 종료, 부드러운 전환 | 15-30f |
| `slideFromLeft` | 왼쪽에서 슬라이드 등장 | 새 단계 시작 | 20-30f |
| `slideFromRight` | 오른쪽에서 슬라이드 등장 | 다음 항목 | 20-30f |
| `slideFromBottom` | 아래에서 등장 | 자막, 카드 | 15-20f |
| `scaleUp` | scale 0 → 1 (확대) | 강조 요소, 아이콘 | 15-20f |
| `scaleDown` | scale 1 → 0 (축소) | 요소 제거 | 10-15f |
| `wipe` | 가로 와이프 전환 | 슬라이드 느낌 | 20-30f |
| `none` (컷) | 즉각 전환 | 빠른 편집, 강조 | 0f |

## 비주얼 요소 카탈로그

씬에서 사용할 수 있는 비주얼 요소 유형.

### 텍스트 요소

- **대제목**: 화면 중앙, 큰 폰트 (48-72px)
- **소제목**: 대제목 아래, 보조 설명 (24-36px)
- **자막**: 화면 하단, 나레이션 표시 (18-24px)
- **강조 텍스트**: 배경 하이라이트 + 볼드

### 코드 요소

- **코드 블록**: syntax highlight 적용, 배경 어두운 색
- **코드 라인 하이라이트**: 특정 줄 강조
- **터미널 출력**: 검정 배경, 초록/흰색 텍스트

### 다이어그램 요소

- **박스 + 화살표**: 데이터 흐름, 아키텍처
- **순서도**: 단계별 프로세스
- **연결선**: 관계 표현

### 미디어 요소

- **이미지/스크린샷**: UI 예시, 참고 자료
- **아이콘**: 개념 시각화
- **화면 녹화 영역**: 실제 동작 데모

### 데이터 시각화

- **막대 차트**: 비교 데이터
- **꺾은선 그래프**: 시계열 데이터
- **원형 차트**: 비율 표현
- **숫자 카운트업**: 통계 강조

### 배경 효과

- **단색 배경**: 깔끔한 집중
- **그라디언트 배경**: 세련된 느낌
- **파티클 효과**: 인트로/아웃트로 임팩트
- **블러 배경**: 텍스트 가독성 향상

## Remotion 컴포넌트 매핑

- **컴포넌트명**: `Scene{N}{제목PascalCase}` (예: `Scene01Intro`)
- **파일 위치**: `src/scenes/Scene{N}-{제목}.tsx`
- **주요 props**:
  ```typescript
  interface Scene{N}Props {
    text?: string;
    code?: string;
    highlightLines?: number[];
  }
  ```
- **사용 애니메이션**: interpolate(fadeIn), spring(scaleUp)

## 16:9 vs 9:16 레이아웃 가이드

### 16:9 (롱폼, YouTube 기본)

- 해상도: 1920 × 1080
- 콘텐츠 영역: 양쪽 여백 80-120px
- 자막 위치: 하단 80-120px
- 코드 블록: 중앙 정렬, 최대 너비 1400px

### 9:16 (숏폼, YouTube Shorts / Reels)

- 해상도: 1080 × 1920
- 콘텐츠 영역: 상하 여백 160-200px
- 자막 위치: 하단 중앙 (손가락에 가리지 않도록)
- 텍스트 크기 1.3-1.5배 증가 (모바일 가독성)
- 핵심 정보는 화면 중앙 1/3 영역에 배치

---

## 주의사항

- 스크립트의 [VISUAL] 힌트를 반드시 레이아웃 테이블에 반영한다
- 비주얼 일관성 확보: 색상, 폰트, 레이아웃을 프로젝트 전체에서 통일한다
- 16:9와 9:16 혼용 금지 — 프로젝트 시작 시 하나를 결정하고 유지한다
- 프레임 계산 오류 방지: 씬 시작 프레임 = 이전 씬들의 durationInFrames 합계
- 에셋 파일명은 소문자, 하이픈 구분으로 통일한다 (예: `scene-01-code.png`)
- 스토리보드는 Remotion 구현 전에 완성해야 한다
