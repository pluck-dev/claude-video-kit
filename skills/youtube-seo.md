# YouTube SEO 최적화

## 개요

YouTube 영상의 검색 노출과 추천을 극대화하는 SEO 전략.
제목, 설명, 태그, 챕터 마커, 썸네일 각각의 규칙을 따른다.

---

## 제목(Title) 규칙

### 기본 원칙

- **최대 60자** (모바일 검색 결과에서 잘리지 않는 안전 길이)
- **핵심 키워드를 앞쪽에 배치** (앞 30자 내에 주요 키워드 포함)
- 클릭을 유도하되 내용과 일치해야 한다 (클릭베이트 금지)

### 제목 강화 패턴

| 패턴 | 예시 | 효과 |
|------|------|------|
| 숫자 활용 | "5가지 React 실수" | 구체성, 신뢰감 |
| 시간 제한 | "10분 만에 이해하는 TypeScript" | 접근성 |
| 연도 태그 | "[2024] Next.js 완벽 가이드" | 최신성 |
| 난이도 태그 | "[초보자용] Git 사용법" | 타겟 명확화 |
| 감정 유발 | "개발자가 꼭 알아야 할 필수 도구" | 클릭률 향상 |
| 결과 강조 | "이것만 알면 면접 통과하는 CS 지식" | JTBD |

### 제목 작성 예시

```
Good:
  "React useState 완벽 정리 | 5가지 실전 패턴 [2024]"
  "10분 만에 배우는 Docker 기초 - 초보자 필수"
  "TypeScript 제네릭, 이제 진짜 이해했습니다"

Bad:
  "리액트에 대해 알아보겠습니다" (키워드 없음, 너무 모호)
  "이 영상 하나로 당신의 인생이 바뀝니다!!!" (클릭베이트)
  "React Hook useState useEffect 리액트 훅 리액트강의" (키워드 남용)
```

## 설명(Description) 규칙

### 구조

YouTube 설명은 처음 2-3줄이 핵심이다. 더보기를 클릭하기 전에 보이는 영역이므로
가장 중요한 정보를 상단에 배치한다.

```
[첫 2줄: 핵심 요약 + 주요 키워드]
[빈 줄]
[챕터 마커]
[빈 줄]
[추가 정보: 관련 링크, 소셜, 리소스]
[빈 줄]
[해시태그]
```

### 키워드 포함 규칙

- 핵심 키워드를 자연스럽게 3-5회 포함한다
- 첫 문장에 주요 키워드 반드시 포함
- 키워드 스터핑(강제 반복 삽입) 금지 — 자연스러운 문장으로 작성
- 관련 키워드(동의어, 관련어)도 포함하면 노출 확대

### 설명 작성 예시

```
이 영상에서는 React useState를 완벽하게 이해하고 실전에서 바로 쓸 수 있는
5가지 패턴을 배웁니다. 초보자도 쉽게 따라할 수 있도록 단계별로 설명합니다.

📌 챕터
00:00 인트로
01:30 useState 기초
04:00 패턴 1: 객체 상태 관리
07:15 패턴 2: 배열 상태 관리
11:00 패턴 3: 비동기 상태 처리
15:30 패턴 4: 커스텀 Hook 분리
19:00 패턴 5: 전역 상태와의 연계
22:00 정리 + 다음 영상 안내

📂 관련 링크
- GitHub 코드: https://github.com/...
- 블로그 글: https://...
- 이전 영상: https://youtu.be/...

📬 소셜
- 인스타: @...
- 트위터: @...

#React #useState #프론트엔드 #리액트강의
```

## 태그(Tags) 전략

### 태그 구성

| 유형 | 수량 | 예시 |
|------|------|------|
| 핵심 키워드 | 3-5개 | "React", "useState", "리액트" |
| 롱테일 키워드 | 5-10개 | "React 초보자", "useState 사용법", "리액트 훅 강의" |
| 채널/시리즈명 | 1-3개 | "내 채널명", "리액트 시리즈" |
| 카테고리 | 2-3개 | "프로그래밍", "웹개발", "프론트엔드" |

### 태그 작성 규칙

- 각 태그는 쉼표로 구분
- 한국어 태그와 영어 태그 혼용 (검색어 확대)
- 경쟁 채널명 태그는 사용하지 않는다
- 총 500자 이내 (YouTube 제한)

### 태그 예시

```
React, useState, 리액트, 리액트 훅, React Hook,
useState 사용법, useState 예제, 리액트 초보자,
React 강의, 프론트엔드 개발, 웹개발, JavaScript,
리액트 강좌, React Tutorial
```

## 챕터 마커(Chapter Markers) 자동 생성

### 규칙

- 스크립트의 씬 분할을 기반으로 챕터를 자동 생성한다
- **첫 챕터는 반드시 `00:00`** (없으면 챕터 기능이 활성화되지 않음)
- **최소 3개**, **최대 10개** 챕터 (너무 많으면 오히려 복잡)
- 각 챕터 간 최소 10초 이상 간격

### 포맷

```
00:00 인트로
01:30 {씬 제목}
04:00 {씬 제목}
...
```

### 씬 길이 → 타임스탬프 변환

```
씬 01: 0s ~ 20s    → 00:00
씬 02: 20s ~ 110s  → 00:20
씬 03: 110s ~ 230s → 01:50
씬 04: 230s ~ 340s → 03:50
씬 05: 340s ~ 390s → 05:40
```

## 썸네일(Thumbnail) 텍스트 규칙

### 기본 규칙

- **텍스트**: 최대 5-7 단어 (한눈에 읽혀야 함)
- **대비**: 고대비 색상 (흰색/노란색 텍스트 + 어두운 배경)
- **폰트 크기**: 72pt 이상 (작은 화면에서도 가독성 확보)
- **해상도**: 1280 × 720 (16:9, YouTube 권장) 또는 1920 × 1080

### 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|---------|
| 텍스트 | 5-7 단어, 핵심 키워드 | 필수 |
| 얼굴 이미지 | 감정 표현 (놀람, 자신감 등) | 권장 |
| 배경 | 단색 또는 주제 관련 이미지 | 필수 |
| 브랜드 색상 | 채널 일관성 | 권장 |
| 화살표/강조 | 시선 유도 | 선택 |

### 색상 조합 예시

```
조합 1 (클래식): 흰 텍스트 + 어두운 파란색 배경
조합 2 (에너지틱): 노란 텍스트 + 검정 배경
조합 3 (전문가): 흰 텍스트 + 짙은 네이비 배경
조합 4 (경고/강조): 빨간 텍스트 + 흰 배경
```

### Remotion 썸네일 생성

```bash
# 특정 프레임을 썸네일로 추출
npx remotion still src/index.ts Thumbnail --frame=0 output/thumbnail.png --width=1280 --height=720
```

## 숏폼 SEO 차이점

YouTube Shorts, Instagram Reels, TikTok은 일반 롱폼과 SEO 전략이 다르다.

| 항목 | 롱폼 | 숏폼 |
|------|------|------|
| 제목 | 최대 60자 | **최대 40자** (더 짧게) |
| 설명 | 상세하게 | 2-3문장으로 간결하게 |
| 해시태그 | 3-5개 | **#Shorts 필수 포함**, 5-8개 |
| 챕터 | 권장 | 불필요 (60초 이내) |
| 가장 중요한 것 | 제목 + 태그 | **첫 3초 후킹** |

### 숏폼 제목 예시

```
Good:
  "useState 5초 정리 #shorts"
  "이 한 줄로 버그 해결 #React"
  "개발자 필수 단축키 #coding"

Bad:
  "React useState 완벽 정리 - 초보자도 쉽게 따라할 수 있는 5가지 패턴" (너무 김)
```

## 주의사항

- 클릭베이트 제목은 시청 지속 시간을 낮춰 오히려 노출이 줄어든다
- 키워드 스터핑은 YouTube 알고리즘 페널티 대상이다
- 채널 전반의 일관된 브랜딩이 구독자 유지에 중요하다
- 영상 업로드 후 첫 24-48시간의 클릭률(CTR)과 시청 지속 시간이 초기 노출을 결정한다
- 챕터 마커는 시청자가 원하는 부분을 바로 찾게 해줘 시청 만족도를 높인다
- 설명의 링크는 youtu.be 단축 링크보다 전체 URL을 사용한다
